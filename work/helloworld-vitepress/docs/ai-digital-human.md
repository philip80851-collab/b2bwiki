# AI 数字人：把专家表达变成可复制的视频资产

AI 数字人适合承接标准化、需要频繁更新的讲解任务。它的核心价值不是替代真人，而是让企业在不反复安排拍摄的情况下，持续输出稳定、统一、多语言的视频内容。

## 适用场景

- **产品讲解**：功能演示、版本更新、常见问题解答。
- **销售支持**：行业方案、客户跟进、会后资料说明。
- **培训交付**：员工入职、渠道培训、客户操作教程。
- **招商获客**：平台招商、加盟政策、活动邀请。
- **国际业务**：同一内容快速生成多语言版本。

不建议把数字人作为高管致辞、危机回应或重大签约等强信任场景的唯一表达者。

## 标准生产流程

1. **确定目标**：明确受众、渠道和希望观众采取的下一步行动。
2. **整理事实**：锁定产品参数、客户案例、可公开数据和禁用表述。
3. **编写口播**：开头提出客户问题，中段给出方案，结尾设置行动指令。
4. **生成与合成**：选择形象和声音，加入字幕、产品画面、录屏及品牌包装。
5. **人工审核**：业务、品牌和合规负责人共同确认后发布。
6. **数据复盘**：根据跳出点、完播率和咨询问题调整脚本模板。

## 一条 B2B 口播的结构

```text
客户痛点（5 秒）
→ 问题造成的业务影响（10 秒）
→ 产品如何解决（20–40 秒）
→ 证据或案例（10–20 秒）
→ 下一步行动（5 秒）
```

避免让数字人连续朗读大段企业介绍。每 5–10 秒切换字幕层级、产品界面、案例数据或场景画面，才能维持信息密度。

## 项目准备清单

| 类别 | 需要准备的内容 |
| --- | --- |
| 品牌 | Logo、字体、颜色、片头片尾、字幕规范 |
| 人物 | 形象授权、声音授权、语气与发音规则 |
| 业务 | 产品资料、目标客户、卖点优先级、常见异议 |
| 素材 | 产品录屏、现场画面、客户案例、数据图表 |
| 审核 | 事实核验人、合规禁区、发布审批流程 |

## 衡量效果

同时看生产效率和业务结果：单条制作时间、单条成本、版本数量、完播率、有效咨询量、销售转发率，以及内容更新所需时间。若播放量增长但没有带来咨询或销售使用，通常说明选题或行动指令需要调整。

## 主要风险

- 获得形象、声音及素材的明确授权，并保存授权范围。
- 对数字人内容作适当标识，避免让受众误认为真人实时表达。
- 产品参数、案例数据和承诺必须人工核验。
- 建立发音词典，重点检查品牌名、人名、数字和专业术语。
- 保存脚本、版本和审核记录，便于后续追溯与更新。

::: tip 最小可行试点
选取销售团队每周都会解释的问题，先制作 3 条 60–90 秒视频，在同一渠道连续发布两周，再决定是否扩大投入。
:::
